In short

I'm a frontend web application designer and engineer, with a preference for JavaScript applications built on top of Node.js, typically using things like SvelteKit, Next.js, GraphQL, and various types of serverless APIs. This is usually used in a decoupled fashion with an appropriate headless CMS and other data sources, leveraging incremental static regeneration, server-side rendering, and proxy API routing with server-side authentication. The result: focused specialisation, high performance, and greater security.
